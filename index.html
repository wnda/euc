What is euc.js?

stabs.js is a JavaScript module for warning your users that your website uses cookies, in order to comply with EU regulations.

The lib is just shy of 2.5 kilobytes in size, or 1 kilobyte when minified.

It’s not something I’m proud of. It’s not the friendliest or the most modern approach.

However, it’s easy to use, you can customise how it works, and it's pretty robust.

Why make this lib?

Bootstrap and other libraries/frameworks tend to require jQuery. Not surprising. My 50 lines of vanilla would probably be about 8 lines of jQuery.

However, it is 2016 and ES5/ES6 are gaining traction, and the ES3 that is actually supported in the real world is now largely standardised even in IE.

There are of course some vanilla JS alternatives, but they all force hash fragments and that's a feature I really didn‘t want.

I wanted something lean and robust.

Thus, stabs.js was born.

How does it work?

stabs.js uses only vanilla JavaScript primarily relying on for loops and classList.

This small lib loops through everything in the DOM and adds event listeners to everything that is a “tab” in a tabcontainer.

These event listeners wait for a tab to be clicked and then, if the tab is not active, it and its corresponding tabpane are given an active CSS class and all other tabs/tabpanes have their active status removed.

This might seem like cheating, but the approach could easily be adapted to manipulate styles directly. However, appending and removing classes is ultimately more efficient, and also enables the exact way tabs are displayed/hidden to be very flexible to the CSS-savvy dev.

This is ultimately the simplest way to work with tabs.

How does one use it?

Include stabs.min.js in your page and below call the function, optionally passing in a configuration object like so:

    stabs({
      tabcontainer: "tabcontainer",
      tab: "tab",
      tabpane: "tabpane",
      active: "active",
      addhash: false
    });

Then just make sure you have elements corresponding to your selectors. Only CSS classes are supported as selectors at the moment, and you should not include the periods/full-stops preceding the class names in your config object, as these are added automatically.

Updates will include accommodation for any and all selectors supported by querySelectorAll. Using querySelectorAll means that the tabs/tabpanes can be structured however you please, as long as they are all within the tabcontainer.

You need a containing element and the tabs and tabpanes must be children of the container, though they need not necessarily be immediate children.

Passing in true for addhash enables hash fragments for your tabs if you want it.

FAQs

Can I nest tabs?No. This library represents the outcome of a choice: flexible markup or amenability to nesting. Nesting can be done simply if you iterate only over sibling elements to tabs, but then your tabs and tabpanes must be at the same level of the DOM. That sucks. You want to be able to use whatever markup you need to use. And if you're nesting tabs you're probably doing something very bad in terms of UI design.
What’s the ol’ IE compatibility like?IE8 due to the use of querySelectorAll. You might think that classList would mean that minimum un-polyfilled support for IE would be IE10, but a (very) rudimentary polyfill for adding/removing CSS classes is included as the amendCss function.
Why classList?I did originally try to keep to just className amends, but the code was rather inefficient. classList is a no-brainer.
